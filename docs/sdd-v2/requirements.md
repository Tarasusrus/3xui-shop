# Requirements: Меню v2 — Клиентский флоу + Админ-панель + 3x-ui

> Итерация v2. Первая итерация (sdd/): payment flow, reminder, referral — реализованы.
> Эта спека: починить меню, подключить 3x-ui, multi-admin.

---

## F-1: Главное меню клиента

**REQ-1.1** При нажатии /start пользователь регистрируется (если новый) и получает триал 3 дня автоматически — без дополнительного клика.

**REQ-1.2** Главное меню содержит ровно 6 кнопок в указанном порядке:
1. «📋 Заполнить профиль (+4д)»
2. «📡 Моя подписка»
3. «💳 Оплатить»
4. «👥 Пригласить друга»
5. «💬 Написать нам»
6. «📖 Инструкции»

**REQ-1.3** Кнопки, не перечисленные в REQ-1.2, не отображаются в главном меню.

---

## F-2: Раздел «Заполнить профиль»

**REQ-2.1** Если пользователь уже заполнял профиль (device + email заданы) — показать сообщение «Профиль уже заполнен» без FSM.

**REQ-2.2** Если профиль не заполнен — запустить FSM: выбор устройства (Android / iPhone / Mac / Windows) → ввод email → начисление +4 дня к подписке.

**REQ-2.3** Бонус +4 дня начисляется **один раз** на аккаунт.

---

## F-3: Раздел «Моя подписка»

**REQ-3.1** Показывает: дата окончания подписки, статус (Активна / Истекла / Триал), кнопка «Продлить».

**REQ-3.2** Показывает историю последних 5 транзакций: дата, метод, статус.

**REQ-3.3** Кнопка «Продлить» переводит в F-4.

---

## F-4: Раздел «Оплатить»

**REQ-4.1** Отображаются только методы СБП и TON. Все прочие методы (Stars, YooKassa, YooMoney, Cryptomus, Heleket) в UI не показываются.

**REQ-4.2** Управление видимостью методов — через env-флаги `SHOP_PAYMENT_SBP_ENABLED` и `SHOP_PAYMENT_TON_ENABLED`. Флаги остальных методов в .env установлены в `False`.

**REQ-4.3** СБП: бот показывает номер `+7 903 918 90 74`, банк `Цифра банк`, сумму `100 ₽`.

**REQ-4.4** TON: бот показывает адрес кошелька `UQA9_RgcaOGpjI7P-cYPBBEHuLXa4rWhdRzO9LecAG84zeDq`, аккаунт `@Tapac1990`, сумму `1 USDT`.

**REQ-4.5** Кнопка «Я оплатил»: если нет PENDING-транзакции — создать `Transaction(PENDING)` + подписка +30д оптимистично. Если есть — показать блокировку.

**REQ-4.6** Одновременно допускается только 1 PENDING-транзакция на пользователя (любой метод).

**REQ-4.7** PENDING-транзакция без подтверждения через 3 дня → `EXPIRED`. Пользователь уведомлён. Подписка не меняется.

---

## F-5: Раздел «Пригласить друга»

**REQ-5.1** Показывает: реферальную ссылку, количество приглашённых, суммарные бонусные дни.

**REQ-5.2** Бонус +30 дней рефереру при каждом **подтверждении** оплаты друга админом.

**REQ-5.3** Реферер получает уведомление о начислении бонуса.

---

## F-6: Раздел «Написать нам»

**REQ-6.1** Запускает FSM: пользователь вводит текст → бот пересылает на `BOT_SUPPORT_ID` с указанием `tg_id` отправителя.

**REQ-6.2** Пользователь получает подтверждение «Сообщение отправлено».

---

## F-7: Раздел «Инструкции»

**REQ-7.1** Кнопка «Инструкции» отображается в главном меню.

**REQ-7.2** При нажатии бот отвечает заглушкой: «Инструкции скоро появятся». Контент — в отдельной спеке.

## F-8: Подключение 3x-ui (диагностика + fix)

**REQ-8.1** Определить причину ошибки «Нет активных серверов»: проверить, есть ли запись в таблице `Server` и доступен ли XUI_HOST.

**REQ-8.2** Через команду /admin_server добавить сервер 3x-ui (172.86.95.214:37521, vpnFran).

**REQ-8.3** После добавления сервера `_try_auto_gift_trial()` и `create_subscription()` должны работать без ошибки «нет серверов».

---

## F-9: Admin Tools — все кнопки рабочие

**REQ-9.1** Команды `/admin_server`, `/admin_stats`, `/admin_users`, `/admin_promos`, `/admin_notify`, `/admin_backup`, `/admin_maintenance` — доступны только пользователям с `is_admin=True` и выполняют заявленную функцию.

**REQ-9.2** Команда `/admin_payments` показывает список PENDING-транзакций (все методы) с полями: username, метод, дата, сумма + кнопки ✅ Подтвердить / ❌ Отклонить.

**REQ-9.3** ✅ Подтвердить → `COMPLETED`; реферальный хук срабатывает; пользователь уведомлён.

**REQ-9.4** ❌ Отклонить → `REJECTED`; вычет ≤30 дней (дата окончания не уходит в прошлое); пользователь уведомлён.

**REQ-9.5** Двойной клик ✅/❌ на уже обработанной транзакции — игнорируется, WARNING в логах.

**REQ-9.6** При отсутствии PENDING-транзакций `/admin_payments` отвечает «Нет ожидающих оплат».

## Нефункциональные требования

**NFR-1** Все реквизиты оплаты — из конфига (env), не хардкодить.

**NFR-2** Методы оплаты, не перечисленные в F-4, не удалять из кода — только отключить флагами.

**NFR-3** Все переходы статусов транзакций логировать с уровнем `INFO`.

**NFR-4** Admin-команды не реагируют на не-админов (silent или «Нет доступа»).

---

## Out of scope

- Контент «Инструкций» (текст, QR, ссылки на приложения)
- Автопроверка СБП/TON через API
- YooKassa, Cryptomus, YooMoney — не удалять
- Промокоды, частичная оплата
- Многоуровневая admin-иерархия
