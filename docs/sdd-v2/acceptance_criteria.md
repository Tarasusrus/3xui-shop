# Acceptance Criteria: Меню v2

---

## AC-1: /start и главное меню

**AC-1.1**
- WHEN: Новый пользователь нажимает /start
- THEN: Регистрация + авто-триал
- SHALL: User создан; подписка +3 дня выдана без дополнительных кликов; главное меню показано

**AC-1.2**
- WHEN: Любой пользователь открывает главное меню
- THEN: Кнопки отображены
- SHALL: Ровно 6 кнопок в порядке: «📋 Заполнить профиль (+4д)», «📡 Моя подписка», «💳 Оплатить», «👥 Пригласить друга», «💬 Написать нам», «📖 Инструкции»

**AC-1.3**
- WHEN: Повторный /start у существующего пользователя
- THEN: Триал не дублируется
- SHALL: Подписка не изменяется; главное меню показано

---

## AC-2: Заполнить профиль

**AC-2.1**
- WHEN: Пользователь уже заполнял профиль (device + email заданы)
- THEN: Нажимает «Заполнить профиль»
- SHALL: Сообщение «Профиль уже заполнен»; FSM не запускается; бонус не начисляется

**AC-2.2**
- WHEN: Профиль не заполнен; пользователь проходит FSM (device → email)
- THEN: FSM завершён
- SHALL: device и email сохранены; подписка +4 дня; подтверждение показано

**AC-2.3**
- WHEN: Пользователь прошёл FSM дважды (edge case)
- THEN: Второй проход
- SHALL: +4 дня **не** начисляется повторно

---

## AC-3: Моя подписка

**AC-3.1**
- WHEN: Пользователь открывает «Моя подписка»
- THEN: Страница показана
- SHALL: Видны дата окончания, статус (Активна/Истекла/Триал), кнопка «Продлить», история ≤5 транзакций

**AC-3.2**
- WHEN: Пользователь нажимает «Продлить»
- THEN: Переход выполнен
- SHALL: Открывается раздел оплаты (AC-4)

---

## AC-4: Оплатить

**AC-4.1**
- WHEN: `SHOP_PAYMENT_SBP_ENABLED=True`, `SHOP_PAYMENT_TON_ENABLED=True`
- THEN: Пользователь открывает «Оплатить»
- SHALL: Видны ровно 2 кнопки: «СБП» и «TON». Stars, YooKassa и прочие — отсутствуют

**AC-4.2**
- WHEN: Пользователь выбирает СБП
- THEN: Страница реквизитов показана
- SHALL: Видны +7 903 918 90 74, Цифра банк, 100 ₽, кнопка «Я оплатил»

**AC-4.3**
- WHEN: Пользователь выбирает TON
- THEN: Страница реквизитов показана
- SHALL: Видны адрес кошелька, @Tapac1990, 1 USDT, кнопка «Я оплатил»

**AC-4.4**
- WHEN: Нет PENDING-транзакции; пользователь нажимает «Я оплатил»
- THEN: Оплата принята
- SHALL: Transaction(PENDING) создана; подписка +30д; сообщение «Оплата принята, ожидаем подтверждения»

**AC-4.5**
- WHEN: Есть PENDING-транзакция; пользователь нажимает «Я оплатил»
- THEN: Блокировка
- SHALL: Новая транзакция не создаётся; показано сообщение о блокировке

**AC-4.6**
- WHEN: PENDING-транзакция не подтверждена 3 дня
- THEN: Фоновая задача срабатывает
- SHALL: Статус → EXPIRED; пользователь уведомлён; подписка не изменяется; кнопка «Я оплатил» снова активна

---

## AC-5: Пригласить друга

**AC-5.1**
- WHEN: Пользователь открывает «Пригласить друга»
- THEN: Страница показана
- SHALL: Видны реферальная ссылка, кол-во приглашённых, суммарные бонусные дни

**AC-5.2**
- WHEN: Админ подтверждает оплату referred-пользователя
- THEN: Хук срабатывает
- SHALL: Рефереру +30 дней; реферер получает уведомление

---

## AC-6: Написать нам

**AC-6.1**
- WHEN: Пользователь нажимает «Написать нам» и вводит текст
- THEN: Сообщение отправлено
- SHALL: Текст forwarded на BOT_SUPPORT_ID с tg_id отправителя; пользователь видит «Сообщение отправлено»

---

## AC-7: Инструкции (заглушка)

**AC-7.1**
- WHEN: Пользователь нажимает «Инструкции»
- THEN: Ответ получен
- SHALL: Текст-заглушка «Инструкции скоро появятся» (или аналог); бот не падает

---

## AC-8: 3x-ui подключение

**AC-8.1**
- WHEN: Сервер добавлен через /admin_server
- THEN: Новый пользователь нажимает /start
- SHALL: Триал выдан без ошибки «нет серверов»; клиент создан в 3x-ui

**AC-8.2**
- WHEN: Сервер добавлен
- THEN: Пользователь оплачивает и получает подтверждение
- SHALL: create_subscription выполнена без ошибки; пользователь получает VPN-конфиг

---

## AC-9: Admin Tools

**AC-9.1**
- WHEN: Не-админ вводит любую /admin_* команду
- THEN: Команда обработана
- SHALL: Бот не реагирует или отвечает «Нет доступа»

**AC-9.2**
- WHEN: Нет PENDING-оплат; админ вводит /admin_payments
- THEN: Ответ получен
- SHALL: «Нет ожидающих оплат»

**AC-9.3**
- WHEN: Есть PENDING-оплаты; админ вводит /admin_payments
- THEN: Список показан
- SHALL: Каждая запись содержит username, метод, дату, сумму + кнопки ✅ / ❌

**AC-9.4**
- WHEN: Админ нажимает ✅
- THEN: Транзакция обработана
- SHALL: COMPLETED; реф. хук срабатывает; пользователь уведомлён

**AC-9.5**
- WHEN: Админ нажимает ❌
- THEN: Транзакция обработана
- SHALL: REJECTED; вычет ≤30 дней (дата не уходит в прошлое); пользователь уведомлён

**AC-9.6**
- WHEN: Транзакция уже COMPLETED или REJECTED; двойной клик
- THEN: Повторный вызов
- SHALL: Действие игнорируется; WARNING в логах

## AC-10: Multi-admin

**AC-10.1**
- WHEN: BOT_ADMINS содержит несколько tg_id через запятую
- THEN: Каждый из них вводит /admin_payments
- SHALL: Оба получают список; оба могут confirm/reject

**AC-10.2**
- WHEN: 403809728 (Evgeniia Furman) вводит /admin_payments
- THEN: Доступ получен
- SHALL: Список PENDING-оплат показан
